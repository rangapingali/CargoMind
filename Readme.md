# 🚀 CargoMind — AI-Powered Supply Chain Intelligence Platform

## 🧠 Overview

CargoMind is a full-stack web application built to **predict and prevent supply chain disruptions before they happen**.

This project was developed as part of a hackathon by **Team Trifecta Tech**, with the goal of moving supply chain systems from reactive problem-solving to **proactive intelligence**.

Instead of analyzing failures after they occur, CargoMind focuses on identifying risks **3–5 days in advance** using multiple real-time signals and AI-powered insights.

---

## ⚡ The Idea Behind CargoMind

Modern supply chains often fail because decisions are taken **after damage is already done**.

We wanted to build something that:

* Detects problems early
* Helps decision-makers act faster
* Reduces losses and delays
* Makes supply chain systems smarter and more reliable

CargoMind is our attempt to solve that.

---

## 🎯 Key Features

### 🔮 Predictive Intelligence

* AI-powered disruption detection
* Real-time disruption scoring system
* Smart alerts with actionable recommendations

---

### 📊 Live Signal Monitoring

CargoMind considers multiple real-world factors like:

* Traffic and logistics conditions
* Weather disruptions
* Fuel and economic changes
* Operational delays and external risks

---

### 👥 Multi-Role System

The platform supports multiple stakeholders with separate dashboards:

* Supply Chain Manager
* Manufacturer
* Logistics Manager
* Truck Driver
* Supplier
* Warehouse Manager
* Retailer
* Healthcare / Pharma

Each role sees only relevant data, keeping the system clean and efficient.

---

### 🧭 Decision Support System

CargoMind follows a structured workflow:

1. Detect disruption
2. Identify affected shipment
3. Suggest possible actions
4. Get user approval
5. Execute decision

This ensures:

* Controlled decision-making
* Accountability
* Better coordination


> Note: You will need to add your own API keys (Firebase, Gemini, Google Maps) in environment variables to run the project locally.
